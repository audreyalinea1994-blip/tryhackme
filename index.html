What is Ethical Hacking?

Ethical hacking (also known as penetration testing or white-hat hacking) is the practice of deliberately probing computer systems, networks, and applications for security vulnerabilities — with the explicit permission of the owner. The goal is to identify and fix weaknesses before malicious hackers can exploit them.

Unlike malicious hackers (black hats), ethical hackers operate within legal boundaries and follow a strict code of conduct. They use the same tools and techniques as attackers but with a defensive purpose.

KEY PRINCIPLE: "To catch a thief, you must think like a thief — but act like a guardian."

Types of Hackers

🤍

White Hat

Authorized security professionals who test systems with permission. They work for organizations or as consultants to improve security defenses.

🩶

Grey Hat

Operate without explicit permission but without malicious intent. They may report vulnerabilities but sometimes violate laws or ethical standards.

🖤

Black Hat

Malicious hackers who exploit vulnerabilities for personal gain, data theft, or disruption. Their activities are illegal and unethical.

📜

Script Kiddie

Inexperienced individuals who use pre-made hacking tools without understanding the underlying concepts. Often cause unintended damage.

✊

Hacktivist

Hackers motivated by political or social causes. Groups like Anonymous use hacking to promote ideological agendas or protest injustice.

🏛️

State-Sponsored

Government-backed hackers conducting cyber espionage, intelligence gathering, or cyber warfare against other nations or organizations.

The 5 Phases of Ethical Hacking

1

Reconnaissance (Footprinting)

Gathering information about the target — domain names, IP ranges, employee info, tech stack. Uses tools like WHOIS, Shodan, Google Dorking, and social media OSINT.

2

Scanning & Enumeration

Actively probing the target for open ports, services, and vulnerabilities. Tools include Nmap, Nessus, Nikto, and OpenVAS. Identifies potential entry points.

3

Gaining Access (Exploitation)

Exploiting discovered vulnerabilities to gain entry. This may involve SQL injection, XSS, buffer overflows, or password cracking using Metasploit, Burp Suite, or Hydra.

4

Maintaining Access

Testing if persistent access can be maintained through backdoors, rootkits, or trojans. Simulates advanced persistent threats (APTs) to test detection capabilities.

5

Covering Tracks & Reporting

Documenting all findings, vulnerabilities, and evidence. Creating a detailed report with risk assessments and remediation recommendations for the client.

Common Attack Vectors

SQL Injection (SQLi)

Inserting malicious SQL code into input fields to manipulate databases. Can extract, modify, or delete data.

' OR '1'='1' --

Cross-Site Scripting (XSS)

Injecting client-side scripts into web pages viewed by other users. Can steal cookies, sessions, or redirect users.

<script>document.cookie</script>

Phishing

Social engineering attack using fake emails, websites, or messages to trick users into revealing credentials or installing malware. #1 attack vector worldwide.

Man-in-the-Middle (MitM)

Intercepting communications between two parties. Attackers can eavesdrop, modify data, or inject malicious content using tools like Wireshark or Ettercap.

Brute Force Attacks

Systematically trying every possible password combination. Dictionary attacks use wordlists. Tools: Hydra, John the Ripper, Hashcat.

DDoS Attacks

Overwhelming a target with massive traffic from multiple sources, causing service unavailability. Botnets are commonly used to amplify attacks.

Legal & Ethical Framework

Always Get Written Permission

A formal agreement (Rules of Engagement) must be signed before any testing begins. This defines scope, methods, timelines, and liability.

Philippine Laws: RA 10175 (Cybercrime Prevention Act)

Covers illegal access, data interference, system interference, misuse of devices, and cyber-squatting. Penalties include imprisonment and fines.

International: CFAA (USA) & Computer Misuse Act (UK)

Unauthorized access to computer systems is illegal worldwide. Even well-intentioned hacking without permission can result in criminal charges.

⚠️ Golden Rule

NEVER test on systems you don't own or have explicit written authorization to test. Use dedicated practice platforms only!
