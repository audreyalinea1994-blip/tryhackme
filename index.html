MODULE 1 // INTRODUCTION

What is Ethical Hacking?

Definition

Ethical hacking is the authorized practice of bypassing system security to identify potential data breaches and threats. Also known as penetration testing or white-hat hacking.

Purpose

To improve the security posture of an organization by finding and fixing vulnerabilities before malicious hackers can exploit them. Think of it as a security stress test.

Types of Hackers

🎩
White Hat

Authorized testers. Work with organizations to improve security legally and ethically.

🖤
Black Hat

Malicious hackers. Exploit systems for personal gain, data theft, or destruction.

🎭
Grey Hat

Operate without authorization but without malicious intent. Legally ambiguous territory.

MODULE 1 // LEGAL FRAMEWORK

Legal & Ethical Boundaries

⚠️ Golden Rule: Always Get Written Authorization

Unauthorized access to computer systems is illegal under laws like the Computer Fraud and Abuse Act (CFAA), RA 10175 (Cybercrime Prevention Act – Philippines), and similar legislation worldwide. Permission is what separates ethical hacking from cybercrime.

✅ Ethical Hacker Must:

  • → Have written authorization (scope, rules of engagement)
  • → Define and respect the scope of assessment
  • → Report all vulnerabilities found to the organization
  • → Respect data privacy – never steal or leak data
  • → Follow responsible disclosure practices
  • → Document everything thoroughly

🚫 Never Do This:

  • ✕ Test systems without explicit written permission
  • ✕ Go beyond the agreed scope
  • ✕ Access, copy, or exfiltrate real user data
  • ✕ Launch denial-of-service attacks without approval
  • ✕ Share findings publicly without authorization
  • ✕ Use discovered access for personal gain

Key Laws to Know

Philippines
RA 10175 – Cybercrime Prevention Act
Covers illegal access, data interference, cybersquatting, identity theft
USA
CFAA – Computer Fraud and Abuse Act
Federal law criminalizing unauthorized computer access
Europe
GDPR + Budapest Convention
Data protection and international cybercrime cooperation
Certification
CEH – Certified Ethical Hacker
Industry standard from EC-Council. Validates ethical hacking skills.
MODULE 2 // FOOTPRINTING

Footprinting & Reconnaissance

Footprinting is the first and most critical phase of ethical hacking. It involves collecting as much information as possible about a target system to find ways to penetrate it. Think of it as the "research phase" before any attack.

"Give me six hours to chop down a tree and I will spend the first four sharpening the axe." – Abraham Lincoln

🔍 Passive Reconnaissance

Gathering information WITHOUT directly interacting with the target. The target has no way of knowing.

  • • WHOIS lookups & DNS records
  • • Google dorking / advanced search
  • • Social media profiling (OSINT)
  • • Job postings analysis (reveals tech stack)
  • • Wayback Machine / cached pages
  • • Public financial records & SEC filings
  • • Shodan searches for exposed devices

🎯 Active Reconnaissance

Directly interacting with the target to gather information. The target MAY detect this activity.

  • • Port scanning (Nmap)
  • • Ping sweeps & traceroutes
  • • Banner grabbing
  • • Social engineering (phishing, pretexting)
  • • DNS zone transfers
  • • Web spidering / crawling
  • • Network mapping & OS fingerprinting

What Information Are We Looking For?

Network Info

Domain names, IP ranges, DNS records, network topology, firewalls, VPNs

System Info

OS types, web server software, application frameworks, open ports, services

Organization Info

Employee names/emails, org structure, office locations, phone numbers, tech stack

MODULE 3 // SCANNING

Scanning & Enumeration

After footprinting, scanning is the active phase where we probe the target to discover open ports, running services, operating systems, and potential vulnerabilities. Enumeration goes deeper – extracting usernames, shares, and detailed service info.

Types of Scanning

📡

Network Scanning

Discover live hosts, open ports, and network topology

• Ping sweeps
• TCP/UDP port scanning
• ARP scanning (LAN)
🔬

Vulnerability Scanning

Automatically identify known vulnerabilities

• CVE matching
• Misconfigurations
• Default credentials
🕵️

Enumeration

Extract detailed info from discovered services

• Banner grabbing
• User/share enumeration
• SNMP/LDAP queries

Common Ports You Should Know

Port Service What It Does Risk
21 FTP File transfer Anonymous access
22 SSH Secure shell Brute force
23 Telnet Remote access (unencrypted!) Plaintext credentials
80/443 HTTP/S Web server Web vulnerabilities
3306 MySQL Database SQL injection, exposure
3389 RDP Remote Desktop Brute force, BlueKeep
MODULE 3 // NMAP

Nmap – The Network Mapper

Nmap is the #1 tool for network discovery and security auditing. Free, open-source, and used by professionals worldwide.

nmap.org ↗ Documentation ↗

Essential Nmap Commands

Basic scan – discover open ports
nmap scanme.nmap.org
Service version detection – what's running on each port
nmap -sV scanme.nmap.org
OS detection – identify the operating system
nmap -O scanme.nmap.org
Aggressive scan – OS, version, scripts, and traceroute
nmap -A scanme.nmap.org
Scan specific ports
nmap -p 80,443,22,21 scanme.nmap.org
Vulnerability scan with Nmap scripts
nmap --script vuln scanme.nmap.org
